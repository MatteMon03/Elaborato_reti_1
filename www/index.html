HTTP -> Hyper-Text Transfer Protocolo

La pagina 404 non esiste quindi porterà ad avere un errore

    Come si può vedere dall'immagine sopra il protocollo HTTP è un protocollo di comunicazione client-server che permette di trasferire informazioni tra un client e un server.
  • Il client invia una richiesta al server per ottenere una risorsa (pagina web, immagine, file, ecc.) I possibili Client sono i browser web, le app mobili e altri software che richiedono dati da un server.

  • Il server elabora la richiesta e invia una risposta al client, che può includere la risorsa richiesta o un messaggio di errore (ad esempio, 404 Not Found) Il server può essere un computer fisico o virtuale che ospita le risorse e gestisce le richieste dei client

Breve spiegazione del protocollo HTTP raffigurato sopra

    HTTP è un protocollo di comunicazione STATELESS, il che significa che ogni richiesta è indipendente dalle altre, il server risponde ma non memorizza la transizione. I messaggi sono di due tipi RICHIESTA o RISPOSTA e sono formati da una intestazione (HEADER) e un corpo(BODY).
  • L'header è composto da informazioni che riguardano la tipologia del messaggio
  • Il body è composto dalle informazioni effettive del messaggio

    • La richiesta è formata da un metodo, un URL e una versione del protocollo. Le richieste HTTP possono essere di diversi tipi, come GET (per ottenere dati), POST (per inviare dati) e PUT (per aggiornare dati). Il server elabora la richiesta e invia una risposta al client, che può includere la risorsa richiesta o un messaggio di errore (ad esempio, 404 Not Found oppure 200 OK) Quindi l'user si interfaccia con il browser tramite un URL (Uniform Resource Locator) che è un indirizzo web che identifica una risorsa su Internet. L'URL è composto da diverse parti, tra cui il protocollo (ad esempio, http o https), il nome del dominio (ad esempio, www.example.com) e il percorso della risorsa (ad esempio, /pagina.html).
    il browser invia una richiesta HTTP al server per ottenere la risorsa specificata nell'URL. Il server elabora la richiesta e invia una risposta HTTP al browser, che a sua volta la restituisce all'user.
